# **Contributing to the Flask \+ Tailwind App**

This document outlines the process for setting up the project on your local machine and contributing to the codebase.  
Our application is built using **Flask** for the backend and **Tailwind CSS v4** for the frontend styling. The development environment is designed to be as frictionless as possible—**you do not need Node.js or npm installed to work on this project.**

## **Prerequisites**

* **Python 3.8+**  
* **Git**

## **Local Setup**

Follow these steps to get the project running on your machine:  
**1. Clone the repository**  
```
git clone <repository-url>  
cd <repository-directory> 
```

**2. Create and activate a virtual environment**  
# Windows  
```
python -m venv .venv  
./.venv\Scripts\activate
```
# Mac/Linux  
```
python -m venv .venv  
source ./.venv/bin/activate
```
**3. Install dependencies**  
```pip install -r requirements.txt```

## **Running the Application**

How you run the app depends on what you are working on.

### **Scenario A: Working on Backend/Logic (No CSS changes)**

If you are only working on Python code, database models, or Flask routes, you don't need to worry about Tailwind at all. The compiled CSS file is already tracked in the repository.  
Simply start the Flask server:  
```python app.py```

*The app will run locally at http://127.0.0.1:5000/.*

### **Scenario B: Working on Frontend/UI (Adding Tailwind classes)**

If you are modifying HTML templates and adding new Tailwind utility classes, you need the Tailwind watcher running in the background to compile your styles.  
**You do not need npm\!** We have provided a script that automatically downloads and runs the Tailwind Standalone CLI for Windows.  
**Terminal 1: Start the Watcher**  
```python watcher.py```

*(Leave this running in the background while you code. It will automatically detect new HTML files and compile the CSS.)*  
**Terminal 2: Start Flask**  
```python main.py```

## **Project Structure**
```
├── app.py                 # Main Flask application file  
├── requirements.txt       # Python dependencies  
├── watcher.py         # Script to auto-download and run Tailwind CLI  
├── templates/             # HTML templates (Tailwind watches these)  
│   └── index.html           
└── static/  
    └── css/  
        ├── input.css      # Tailwind input (do not edit unless adding custom CSS directives)  
        └── main.css       # Auto-generated CSS file (DO NOT edit manually)
```
## **Important Guidelines**

1. **Never edit static/css/main.css manually.** This file is entirely auto-generated by Tailwind. Any manual changes will be overwritten the next time the watcher runs.  
2. **Commit your CSS changes.** If you ran the watcher and modified the UI, make sure to include static/css/main.css in your Git commit. This ensures that developers who aren't running the watcher (Scenario A) still see your updated UI.  
3. **Keep it modular.** Try to use Tailwind's utility classes directly in the HTML as much as possible rather than writing custom CSS in input.css.

## **Submitting Changes**

1. Create a new branch for your feature or bugfix: `git checkout -b feature/your-feature-name`  
2. Make your changes and test them locally.  
3. Commit your changes with clear, descriptive commit messages.  
4. Push to your branch: git push origin feature/your-feature-name  
5. Open a Pull Request (PR) describing the changes you made.
